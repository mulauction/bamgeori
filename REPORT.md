## 검증 방식과 한계 (정직 보고)

- 매 커밋: `node --check` 전 20+개 모듈 + `esbuild` 전체 번들 해석 통과 → 한국어 커밋 → push → **GitHub Actions에서 `npm run build` 성공 후 Pages 배포**.
- P1 통계 로직은 실제 store+economy를 Node로 구동한 단위 테스트 전 항목 통과(파산/재기/연승/연패/최고잭팟/노동수입).
- **한계:** M1 8GB 커널 패닉 방지 규칙상 로컬 `npm run dev`를 실행하지 않았다. 즉 **3D 렌더링·터치·연출의 실제 화면 동작은 로컬에서 눈으로 확인하지 못했다.** 정적 게이트+CI 빌드는 문법·임포트·번들만 보증하며, Three.js 런타임 동작/레이아웃은 보증하지 않는다. 기존에 배포되어 동작이 확인된 `street` 씬과 동일 패턴(BoxGeometry·Lambert·pixelRatio 캡)을 재사용해 위험을 낮췄다. **최종 확인은 아침 폰 테스트가 담당한다.**

## ☀️ 아침에 폰으로 확인할 테스트 목록

### 기본
1. 라이브 URL 접속 → 거리가 뜨고 조이스틱·시점 드래그·점프가 동작하는가.
2. 화면 아무 곳이나 한 번 탭 후 소리가 나는가(앰비언스/발소리). 우상단 🔊 토글로 음소거·복구 되는가.
3. 거리에 **행인 몇 명이 왔다 갔다** 하는가.

### 3D 미니게임 (핵심 — 각 게임 입장→플레이→결과)
4. **야바위:** 컵 3개가 3D로 섞이는가 → 컵을 **탭**하면 선택되고 들려서 구슬이 보이는가 → 승/패 정산되는가.
5. **개경주:** 개 고르고 출발 → 개 4마리가 달리고 카메라가 선두를 따라가는가 → 결승 슬로우모션 후 정산.
6. **닭싸움:** 닭 고르고 시작 → 두 닭이 돌진·타격하고 깃털·화면 흔들림 → HP바 감소 → 정산.
7. 각 게임 중 **뒤로가기(거리로)** 눌러도 멈추지 않고 정상 복귀하는가(메모리/렉 없이 여러 판 반복).

### 연출·경제
8. 승리 시 **포인트 카운트업**, 큰 배수(개경주 복실이 ×7 등)에서 **빅윈/잭팟 연출**(파티클·진동·암전)이 뜨는가.
9. 잭팟 후 거리로 나오면 **상단 전광판**에 문구가 흐르는가.
10. 대리운전으로 여러 판 → **파산(잔액 0)** 만들기 → 화면이 어두워지고 캐릭터가 남루해지고 걸음이 느려지는가. 승부 가게 입장 버튼이 **"빈털터리 출입 금지"**로 막히고 대리운전만 되는가.
11. 대리운전으로 **5,000P 회복** → 밝아지고 **재기 훈장 토스트**가 뜨는가.
12. **전당포**에서 자전거/중고차/스포츠카 구매 → 거리에 **주차된 탈것**이 나타나는가. 아파트 이상 구매 → 캐릭터 목에 **금목걸이**가 보이는가.
13. **새로고침** 후 포인트·자산·통계·음소거 설정이 유지되는가. (콘솔에서 `__bamgeori.stats()`로 기록 확인 가능)

### 성능
14. 중급 폰에서 3D 게임·거리가 **끊김 없이(체감 30fps↑)** 도는가. 장시간/반복 플레이 시 점점 느려지지 않는가(누수 체크).

## 알려진 리스크 / 다음 세션 후보
- 3D 게임 3종의 실제 조작·연출은 위 폰 테스트 전까지 미검증. 문제 발견 시 해당 게임만 `legacy2d.js`로 즉시 롤백 가능(2D판 보존됨).
- 금목걸이는 별도 상품이 없어 **부동산 등급 보유(apt/bld/statue) 시 표시**로 매핑함(economy WARES 미변경). 별도 목걸이 상품 추가는 기획 결정 필요.
- 옵션 폴리시(야바위 판돈별 셔플 속도, 개경주 배당판 정리) 미착수.
- 확률·배당·정산 로직은 P1~P6 내내 **변경 없음**(수치는 전부 `economy.js`).
