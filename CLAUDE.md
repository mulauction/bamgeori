# 밤거리 승부 (Night Street Hustle)

미니게임 승부·자산 경제·과시 시스템 기반 3D 복셀 게임. 1인 개발.
상세 설계는 루트의 `밤거리승부_기획서_v0.1.docx` 참조 (11개 장: 승패 3층 구조, 경제, 중독 설계, 과시, 소셜, 심의 전략, 로드맵).
초기 레퍼런스 구현은 `bamgeori-3d-v2.html` (Three.js 단일 파일 데모) 참조.

## 절대 원칙
- 포인트(P)는 현금화·환전 절대 불가. 현금 결제 요소는 과시(외형) 아이템 전용이며 판돈으로 전환 불가. 이 원칙을 위반하는 코드는 어떤 경우에도 작성하지 않는다.
- 승부 결과는 실제 확률로만 결정한다. 니어미스(아깝게 지는 연출)를 결과 조작으로 만들지 않는다. 접전 연출은 이미 확정된 결과를 보여주는 방식에서만 허용.
- 로직 먼저, 그래픽 나중. 에셋은 플레이스홀더(코드 생성 복셀)로 만들고 교체 가능한 구조를 유지한다.

## 기술 스택
- Vite + Three.js + vanilla JS (ES modules). 프레임워크 없음.
- 상태: `src/core/store.js` 단일 스토어. 저장: localStorage (추후 서버 전환 대비 save/load 인터페이스 분리).
- 모바일 우선: 터치 조작(가상 조이스틱 + 시점 드래그 + 점프), 세로/가로 대응.

## 폴더 구조
```
src/
  core/      # store(포인트·자산·저장), economy(배당·확률), audio
  street/    # 3D 거리: 씬, 캐릭터, 건물, 조명, 조작
  games/     # 미니게임 1종 = 1폴더 (yabawi/, dograce/, cockfight/, work-daeri/)
  ui/        # 베팅 패널, 결과 연출, 토스트, 상점
  fx/        # 승리 연출 등급제(일반/빅윈/잭팟), 파티클, 화면 효과
assets/      # 추후 상용 복셀 에셋 교체 지점. 경로 하드코딩 금지
```

## 미니게임 공통 인터페이스
모든 게임 모듈은 동일 인터페이스를 구현한다:
```js
export default {
  id, name, minBet,
  mount(container, ctx),   // ctx = { store, fx, audio }
  start(bet),              // Promise<{win: boolean, multiplier: number}>
  unmount()
}
```
정산(포인트 증감)은 게임 모듈이 직접 하지 않고 core/economy가 일괄 처리한다.
승리 연출은 fx 모듈이 배당 배수 기준으로 등급(일반 <5배 / 빅윈 5~10배 / 잭팟 10배+)을 자동 판정한다.

## 작업 방식
- 매 작업 단위마다 `npm run dev`로 브라우저 확인 가능한 상태를 유지한다.
- 커밋은 작게, 한국어 커밋 메시지.
- 확률·배당 수치는 `src/core/economy.js` 한 곳에만 둔다 (밸런싱 시 한 파일만 수정).
- UI 텍스트는 전부 한국어. 코드 주석도 한국어.
